# Support

Use the repository's issue forms for reproducible bugs, case proposals, and feature requests. Use [GitHub Discussions](https://github.com/Dhi13man/harness-evals/discussions) for setup questions, evaluation design, provider-adapter ideas, and interpretation of non-sensitive results.

Before requesting help, run the offline unit tests, record the exact package and provider versions, reduce the failure to the smallest suite or case selection, and remove credentials, private holdout content, absolute home paths, and proprietary source.

Security vulnerabilities must follow [SECURITY.md](SECURITY.md). The project is maintained on a best-effort basis and has no commercial support or response-time guarantee.
